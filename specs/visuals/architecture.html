GooseGrade v2 · System Architecture

Managed app, one deliberate slice of AWS

The product runs on Vercel + Supabase — accounts, per-user data, and row-level security come essentially free. Only the scraper and the multi-agent QA runner reach for Docker and Kubernetes, the parts that are genuinely batch and parallel. Nothing user-facing depends on the cluster.

02 The AWS slice — Docker + Kubernetes

AWSScraper & multi-agent QA kind / minikube local · short-lived EKS demo
Scraper Job — outline.uwaterloo.ca Claude extraction → GradingRule[] QA Job — validate every course Source outlines → Storage

K8s indexed Jobs shard ~1,600 courses across pods — genuine horizontal scaling, not theatre. The QA swarm runs the pure engine over every course and flags any rule that could turn a passing grade into a fail.

Runs ephemerally — ~$0 standing cost. Writes the catalog with the Supabase service role; the app keeps working even if the cluster is down.
